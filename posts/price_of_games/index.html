Price of Games

   2015-06-24      Web Apps

Background

Lets get straight to the point, why pay more for PC games then you should? Pirate or not, it always feels better knowing you supported a developer, even if its minimal. Price of Games aimes to provide you with up to date prices of games across multiple stores/vendors, so you can get the best price on your new games.

Architecture and History

As I decided to dive into webapp development blindly and without direction in an attempt to learn quickly, the initial design decisions for POG are not ideal by any stretch. POG is integrated with Googles Cloud Platform, integrating their app deployment platform and cloud SQL services. POG uses the python tool scrapy, combined with selenium for javascript support, to scrape prices and relevant information from suported sites. For the web-app itself, POG uses Node.js and Express for app services with EJS for rendering HTML. Other tools include Knex and Psycopg2 to interface with the Google Cloud SQL API.

The initial thought for architecture of POG was to keep it contained to the Python environment, using Django for the web side. I quickly realized however, that Django was just not the route I wanted to go for POG and I took the time to learn JS.

In hindesight, all functionality of this app could be implemented solely using JS with a modified MEAN stack. Much of the dificulty and continuing issues associated with POG is due to the over complication of mix matching languages. As JS is required to render the site pages anyway, Selenium and Python is essentially adding more steps to the process than necessary.

Current State

As of writing this page, Price of Games is live. Much of the web side is complete and functions as expected, although current HTML is very minimal and will need to be updated before I consider it complete. The Python/Scrapy data collection tool still has some bugs surrounding inconsistancies in collecting correct information, and the method for scheduling the database update task is currently me remembering to switch it on before bed but this should be an easy fix. Also further tweaking of the SQL proxy authentication is needed to keep the Cloud SQL connection alive for extended periods of time. The current source code is hosted on the GitHub repo.

Things to Do

Apart from addressing the current and ever growing list of bugs and inept HTML, further flushing out of features and site support are essential. The plan is to include Humble Bundle, and potentially resell markets such as G2A in the site search. Features such as game filtering, ordering, advanced searches, and game suggestions are all feasible and should be implemented. More complex jobs such as statistics tracking, price histories, and a live update pages are things I may want to add in the future but current time constraints do not facilitate working on them.
